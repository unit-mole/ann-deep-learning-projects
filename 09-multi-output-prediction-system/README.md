A mixed-output artificial neural network that learns one shared customer representation and predicts **churn probability**, **customer lifetime value**, and **engagement score** simultaneously.

> **Portfolio positioning:** An advanced tabular deep-learning project demonstrating multi-task learning, output-specific heads, imbalanced classification, regression, reproducible preprocessing, and deployable batch inference.

![Architecture](images/model_architecture.png)

## Business Problem

Customer decisions are rarely based on one outcome. Retention teams need churn risk, commercial teams need expected customer value, and customer-success teams need engagement signals. Training isolated models duplicates infrastructure and ignores relationships shared across these outcomes.

This project answers:

> Given one customer profile, can a shared ANN produce multiple related business predictions in a single inference call?

## Model Outputs

| Output | Type | Activation | Loss | Main evaluation |
|---|---|---|---|---|
| Churn probability | Binary classification | Sigmoid | Binary cross-entropy | F1, ROC-AUC, PR-AUC, precision, recall |
| Customer lifetime value | Regression | Linear | Huber during clean retraining | MAE, RMSE, R² |
| Engagement score | Regression | Linear | Huber during clean retraining | MAE, RMSE, R² |

The project is therefore a **mixed-output prediction system**: one classification output and two regression outputs.

## Current Held-Out Benchmark

The bundled artifact was evaluated on 3,000 held-out synthetic rows. The churn rate is only **2.17%**, so accuracy is not used as the primary classification metric.

| Output | Metric | Score |
|---|---:|---:|
| Churn | ROC-AUC | 0.819 |
| Churn | PR-AUC | 0.092 |
| Churn | Precision | 0.122 |
| Churn | Recall | 0.246 |
| Churn | F1 | 0.163 |
| CLV | MAE | 886.75 |
| CLV | RMSE | 1,148.57 |
| CLV | R² | 0.904 |
| Engagement | MAE | 5.30 |
| Engagement | RMSE | 6.68 |
| Engagement | R² | 0.806 |

The regression heads perform strongly. The classification head ranks churn risk reasonably well but remains difficult because of extreme class imbalance. Recommended next experiments include focal loss, calibrated probabilities, precision-at-k, and business-cost threshold selection.

## Important Technical Corrections

The supplied notebook was carefully reviewed and retained in `notebooks/original_supplied_notebook.ipynb`. The portfolio version corrects the following issues:

1. **Target leakage removed.** The original `customer_segment_cluster` used CLV and engagement—the prediction targets—to create an input feature. The production feature set no longer includes that cluster.
2. **Threshold selection moved to validation data.** The original notebook explored thresholds on the test set. The clean training pipeline selects the churn threshold using validation F1 and evaluates once on the test set.
3. **Deployment artifacts added.** The preprocessor, target scaling metadata, input schema, and threshold are saved with the model.
4. **Batch-safe feature engineering added.** Tenure cohorts and spend bands are generated consistently during training and inference.
5. **Model loading made reproducible.** The demo uses a 42-dimensional model artifact aligned exactly with the saved preprocessor.
6. **Evaluation expanded.** PR-AUC, residual plots, output-specific metrics, sample predictions, and model limitations are documented.

## Workflow

```text
Raw customer features
        ↓
Schema validation and engineered bands
        ↓
Median / most-frequent imputation
        ↓
Standard scaling + one-hot encoding
        ↓
Shared ANN representation: 256 → 128 → 64
        ↓
 ┌──────────────────┬──────────────────┬────────────────────┐
 │ Churn head       │ CLV head         │ Engagement head    │
 │ sigmoid          │ linear           │ linear             │
 └──────────────────┴──────────────────┴────────────────────┘
        ↓
Risk buckets, value bands, engagement bands, recommended action
```

## Dataset

The project uses a reproducible **synthetic customer analytics dataset** with 15,000 records. This keeps the repository safe to share while preserving realistic preprocessing challenges:

- numeric and categorical customer attributes;
- controlled missing values;
- a highly imbalanced churn target;
- continuous CLV and engagement targets;
- correlated business outcomes.

The full dataset is generated by `src/data_generation.py` and is intentionally not committed. See [data/README_data.md](data/README_data.md).

## Run Locally

```bash
cd 09-multi-output-prediction-system
python -m venv .venv
```

Windows Command Prompt:

```bat
.venv\Scripts\activate
python -m pip install --upgrade pip
pip install -r requirements.txt
streamlit run streamlit_app.py
```

macOS / Linux:

```bash
source .venv/bin/activate
python -m pip install --upgrade pip
pip install -r requirements.txt
streamlit run streamlit_app.py
```

The app opens at `http://localhost:8501`.

## Retrain the Leakage-Safe ANN

The bundled model is ready for inference. To retrain the architecture from scratch:

```bash
python -m src.model_training --epochs 40
```

This overwrites the deployable artifacts in `models/` and refreshes `outputs/model_metrics.json`.
