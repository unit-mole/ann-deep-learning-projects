# Data Guide

This repository uses a **reproducible synthetic retail pricing dataset** because the original project did not include a public, production-grade commercial pricing dataset.

## Included files

- `sample_input.csv`: small inference-ready file used by the Streamlit batch demo.
- `sample_training_data.csv`: 250 illustrative rows generated by `src/data_generation.py`.

The full training dataset is generated locally and is not committed. This keeps the repository lightweight and avoids implying that synthetic observations are real customer transactions.

## Required inference columns

`product_id` is optional. The remaining fields are required:

```text
day_of_year, weekend_flag, holiday_flag, category, region, channel,
promotion_flag, competitor_promo_flag, base_cost, current_price,
competitor_price, rating, inventory_level, marketing_index, demand_index,
customer_income_index, historical_sales
```

Allowed categories: `beauty`, `electronics`, `fashion`, `grocery`, `home`, `sports`.
Allowed regions: `east`, `north`, `south`, `west`.
Allowed channels: `marketplace`, `online`, `store`.

## Generate a full training dataset

```bash
python -c "from src.data_generation import generate_synthetic_pricing_data; generate_synthetic_pricing_data(16000).to_csv('data/generated_training_data.csv', index=False)"
```

`data/generated_training_data.csv` should remain local unless you intentionally want to version it.

## Replacing synthetic data

For a real implementation, map historical product-price observations to the same decision-time schema and add an observed demand target named `realized_demand`. Do not use future outcomes or an already calculated optimal price as model inputs.
