# Data Guide

This portfolio version uses a **deterministic synthetic banking dataset** generated by `src/data_preprocessing.py`. It contains 25,000 applicant records and was designed to demonstrate an imbalanced binary credit-risk problem without publishing sensitive customer data.

## Included file

- `sample_input.csv`: five safe, non-sensitive example applicants for the Streamlit app.

## Training with another dataset

A replacement training CSV must include the target column `default_flag` and the raw applicant fields listed below:

`age`, `annual_income`, `loan_amount`, `interest_rate`, `employment_length`, `credit_history_years`, `revolving_utilization`, `dti`, `delinquency_count`, `inquiry_count`, `open_accounts`, `existing_loans`, `collateral_value`, `home_ownership`, `purpose`, `grade`, `region`, `verification_status`.

The pipeline derives `monthly_income`, `loan_to_income`, and `collateral_ratio`. Keep private, regulated, or personally identifiable credit data outside Git. The root `.gitignore` excludes common raw-data directories and large data formats.
